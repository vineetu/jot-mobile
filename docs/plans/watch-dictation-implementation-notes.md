## Known prerequisites for tomorrow

1. **Install the watchOS 26 simulator runtime** in Xcode ‚Üí Settings ‚Üí Components. Without it, `xcodebuild` can't build the JotWatch target and the watch app can't be tested in the simulator.
2. **Configure code signing for `JotWatch` and `JotWatchWidgets`** in Xcode. xcodegen sets `DEVELOPMENT_TEAM`, but provisioning profiles for new bundle IDs (`com.vineetu.jot.mobile.Jot.watch` and `‚Ä¶watch.widgets`) need to be auto-generated by Xcode the first time the targets are built.
3. **Embed the watch app in the iOS IPA** ‚Äî this is the missing piece for TestFlight distribution to your watch. Currently the iOS scheme only builds the iOS target. For the watch app to ride along in the iPhone IPA (which is how Apple delivers watch apps), the iOS target needs `JotWatch` listed as a dependency with `embed: true` in `project.yml`. This wasn't added tonight because the watch target couldn't be verified compiling first ‚Äî once you confirm the watch target builds, add:
   ```yaml
   targets:
     Jot:
       dependencies:
         - target: JotWatch
           embed: true
   ```
   Then regen + re-ship.

## Review history

Both design and implementation went through adversarial review cycles:

- **Round 1 design review** found 20 issues (invented blue palette, `.glassEffect()` on watchOS, emoji vs SF Symbols, RootView too tall for 40mm, accidental-Cancel data loss, 5-min silent stop, `updateApplicationContext` coalescing, etc.). All addressed in v2 of the design doc.
- **Round 2 design review** found 4 more blockers (`JotDesign.swift` UIKit import not watch-compatible, schema V5 disagreement between design + plan docs, missing `WKExtendedRuntimeSession`, wrong AOD signal). All addressed in v3 of the design doc + reflected in the implementation.

Implementation-time fixes:
- Swift 6 strict concurrency: `[String: Any]` is not Sendable, so the `WCSession` delegate methods extract typed Sendable values before crossing the actor boundary.
- Cross-process notification observer: subscribed via existing `CrossProcessNotification.addObserver` (Darwin notify center), not `NotificationCenter.default`.
