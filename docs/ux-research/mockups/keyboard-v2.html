Jot iOS — Keyboard Mockup

V2 · Mic + Status Zone Wispr-shape

Mic centered, with a Liquid-Glass status zone above it that morphs across states. Idle shows guidance copy. Recording shows the ✕ / waveform / ✓ shape borrowed from Wispr Flow's recording controls — pure cancel, live amplitude, confirm. Selection shows the 3 prompt chips. The point: the keyboard tells the user where they are in the flow without relying on the Dynamic Island or app foreground. Status copy borrows from Superwhisper's discipline (model-aware, never generic). Liquid Glass is applied ONLY to the status zone — that's where translucency and depth carry meaning, not on every key.

V2 trade-offs:

  • Pro: in-keyboard state feedback — user never wonders "is it recording?" Reduces dependence on Live Activity / Dynamic Island.
  • Pro: the ✕ / waveform / ✓ shape gives users a real cancel-without-paste path that V1 lacks.
  • Pro: needs container→keyboard amplitude IPC (per `keyboard-status-bar-design.md`); bigger build surface.
  • Con: mic-doubles-as-stop AND zone has a stop button — duplicated affordance; needs a clear "primary stop = mic, escape hatch = ✕" interaction story.
