Jot iOS — Keyboard Mockup

V3 · Mic + Accessory Bar Tejas-style

Mic-only below a thin Liquid-Glass accessory bar at the top. The bar hosts Jot-specific affordances: a paste-fresh-transcript pill (with snippet preview), history glyph, settings glyph, and a wand glyph that opens the prompt popover when a selection exists. Closer to what Tejas built (`KeyboardAccessoryBar.swift`) but with no QWERTY committed below it. The point: future-proofs Tier-2 ambitions (paste pill + history) without locking Tier-3 in. Mic placement borrows from Wispr Flow; the monochrome glyph row borrows from Superwhisper's restraint.

V3 trade-offs:

  • Pro: reuses Tejas's `KeyboardAccessoryBar.swift` pattern + `HistoryOverlay.swift` infrastructure — engineering velocity is highest.
  • Pro: the paste pill solves a real "I want to re-insert that transcript" use case the other variants don't address.
  • Con: three competing affordances on idle (paste, history, settings) plus the wand on selection — more visual debt; risks "what does this glyph do?" discoverability problems.
  • Con: popover for prompts is more interaction-heavy than a segmented row directly visible — extra tap to see the choices.
